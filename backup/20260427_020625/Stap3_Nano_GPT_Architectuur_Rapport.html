🤖 Stap 3: Nano-GPT Transformer Architectuur

Het bouwen van een Nederlands Taalmodel - 35 miljoen parameters

📋 Wat is Stap 3?

In Stap 2 hebben we een tokenizer gebouwd die Nederlandse tekst in kleine stukjes (tokens) verdeelt. Nu bouwen we in Stap 3 het neurale netwerk dat zal leren van die tokens.

Dit netwerk heet Nano-GPT en is gemodelleerd naar GPT (Generative Pre-trained Transformer), maar veel kleiner zodat we het kunnen trainen op een normale computer.

❓ De Kernvraag: Waarom Exact Dit Ontwerp?

Drie Belangrijke Beperkingen

1. Dataset Grootte: We hebben ~600 MB gepreprocessed Nederlands tekst (50.000 artikelen)

2. Overfitten: Te veel parameters → het model onthoudt data in plaats van leren

3. Training Snelheid: Meer parameters = langere training (uren in plaats van minuten)

Regel: Parameters ≈ Dataset grootte ÷ 2000. Voor 600 MB: 600.000.000 ÷ 2000 = 300.000 → ~35M parameters is ideaal.

🔍 Inzicht in Elk Onderdeel

1️⃣ Embedding Lagen

Token Embeddings: Zet elk token-ID om in een vector van 512 getallen.

Voorbeeld: Token "is" (ID 312) wordt [0.45, -0.12, 0.78, ..., 0.33] (512 getallen). Deze getallen leren wat "is" betekent in relatie tot andere woorden.

Position Embeddings: Voegt positie-informatie toe zodat het model weet dat het eerste woord "plaats 0" is, het tweede "plaats 1", etc.

Samen (token + positie): "is op plaats 2" is anders dan "is op plaats 5".

2️⃣ Transformer Blok (8× herhaald)

Multi-Head Attention (8 heads):

Dit laat het model "kijken" naar alle tokens tegelijk en bepalen welke relevant zijn.

• Head 1 vraagt: "Welke woorden zijn grammaticaal gerelateerd?"
• Head 2 vraagt: "Welke woorden lijken semantisch op elkaar?"
• Head 3 vraagt: "Welke woorden duidelijker een substantief?"
• enz...

Elk "head" werkt onafhankelijk, dan worden alle antwoorden samengevoegd.

Feedforward Network (2-lagen):

Na attention wordt elke token verder verwerkt:
512 dimensies → 2048 (uitbreiden) → GELU activering → 512 (samentrekken)

Dit is zoals een "verdiepingseenheid" die elk token verder transformeert gebaseerd op wat de attention-laag ervan heeft geleerd.

Residual Connections & Layer Normalization:

Na elk onderdeel (attention en feedforward): x = norm(x + output)
Dit helpt het model beter trainen door "bypass routes" te creëren en waarden stabiel te houden.

3️⃣ Output Laag (Language Model Head met Weight Tying)

De laatste 512-dimensionale vector wordt omgezet naar 16.000 kansen (één per token).

Modern Design (Weight Tying): In plaats van een aparte output matrix (8.2M parameters), gebruiken we de getransponeerde Token Embeddings matrix. Dit bespaart enorm veel parameters zonder verlies van capabiliteit.

Voorbeeld output voor "Dit is een":

"Nederlands" → 23% kans
"mooi" → 15% kans
"bekende" → 12% kans
"geheim" → 0.01% kans
... (16.000 tokens totaal)

Deze kansen bepalen wat het model als volgende woord voorstelt.

🔐 Causal Masking: Het Geheim van Autoregressive Training

Het Problem: In training moeten we voorkomen dat tokens naar de TOEKOMST kijken.

Als we "Dit is een Nederland" trainen, mag het model bij "Nederland" niet al naar het daarna volgende woord kijken. Dan is het vals trainen.

De Oplossing: Causal Mask

Pos 0 ("Dit"): mag naar pos 0 kijken
Pos 1 ("is"): mag naar pos 0-1 kijken
Pos 2 ("een"): mag naar pos 0-2 kijken
Pos 3 ("Nederland"): mag naar pos 0-3 kijken
↓
Elk woord mag ALLEEN naar woorden vóór zich kijken.

In code: een driehoeksmatrix van -inf waarin de toekomst "afgeschermd" is.

🎓 Educatief Inzicht: Waarom Transformers?

Historische Context:

Vóór 2017 gebruikten we RNN's/LSTM's: verwerkt woord-na-woord sequentieel. Probleem: kan niet echt "teruggaan" naar eerdere woorden.

Transformer Doorbraak (2017): "Attention is All You Need"

Idee: elk woord kan DIRECT naar alle andere woorden "kijken" tegelijk. Dit is veel krachtig en kan beter parallelliseren op GPU's.

Multi-Head Attention:
In plaats van één "attention mechanism", hebben we 8 kleine mechanisms die elk een ander aspect van relaties zien. Dit helpt het model informatie uit verschillende hoeken te verwerken.

Waarom Dit Werkt:
Nederlands is vol lange-afstandafhankelijkheden. "Een Nederlands student die veel studeert, wordt slim." Het woord "studeert" is ver weg van "wordt", maar ze zijn verbonden. Transformers kunnen dit pakken!

🔧 Technische Details

class TransformerBlock(nn.Module): def __init__(self, hidden_size, num_heads, intermediate_size, dropout=0.1): self.attention = nn.MultiheadAttention( embed_dim=hidden_size, num_heads=num_heads, dropout=dropout, batch_first=True ) self.feedforward = nn.Sequential( nn.Linear(hidden_size, intermediate_size), nn.GELU(), nn.Linear(intermediate_size, hidden_size), nn.Dropout(dropout) ) def forward(self, x, causal_mask=None): # Attention met residual connection attn_output, _ = self.attention(x, x, x, attn_mask=causal_mask) x = self.norm1(x + attn_output) # Feedforward met residual connection ff_output = self.feedforward(x) x = self.norm2(x + ff_output) return x
Dropout: Willekeurig "uitschakelen" van 10% van de neuronen tijdens training. Dit helpt overfitten voorkomen door het netwerk meer robuust te maken.

GELU Activering: Gelijkwaardig aan ReLU, maar vloeiender. Helpt betere gradiënten tijdens terugpropagatie.

Layer Normalization: Genormaliseert waarden zodat ze mean 0 en std 1 hebben. Stabiliseert training.
