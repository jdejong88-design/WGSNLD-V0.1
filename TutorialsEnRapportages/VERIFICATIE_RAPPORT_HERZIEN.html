❌ Fouten die ik heb Gemist

Fout 1: Merge-Extractie Code is Ongeldig (KRITIEK)

Locatie: 02_train_bpe_tokenizer.py, regels 139-143 (origineel)

Het probleem:

for merge in tokenizer.model.merges:
    f.write(f"{merge[0]} {merge[1]}\n")

Dit aanvaarden dat tokenizer.model.merges een list van tuples is met [0] en [1] indexing. Echter, in de Hugging Face bibliotheek bevat BPE.merges typisch strings (bijv. "Ġ a"), niet tuples. Dit zal een runtime error geven: TypeError: string indices must be integers

Status: GEREPAREERD

Fout 2: Vocab-Extractie Logic Verwarrend (MEDIUM)

Locatie: 02_train_bpe_tokenizer.py, regel 130

Het probleem:

vocab = tokenizer.get_vocab()
merges = tokenizer.get_vocab(with_added_tokens=False)

De tweede variabele merges is niet eens merges — het is gewoon vocab zonder special tokens. Dit is verwarrend en leidt nergens toe.

Status: GEREPAREERD (variabele verwijderd)

Fout 3: Nederlands Taalfout in Print-Statements (TAAL)

Locatie: 02_train_bpe_tokenizer.py, regels 172, 196, 212

Het probleem:

  • print(f"\nVocab size: ...") — Engels. Moet "Vocabulaire grootte" zijn.
  • print(f"Gemiddeld tokens per zin") — "tokens" moet "aantal tokens" zijn.
  • print("[OK] Vocab size: 16,000 tokens") — Mix Engels/Nederlands.

Status: GEREPAREERD

Fout 4: [UNK] Token Opgegeven in Code (CONCEPTUEEL)

Locatie: 02_train_bpe_tokenizer.py, regel 75 (origineel)

Het probleem:

special_tokens=['<|endoftext|>', '[PAD]', '[UNK]']
Byte-Level BPE heeft geen [UNK] token nodig omdat elk karakter opgebroken kan worden in zijn 256 mogelijke byte-waarden. Het is conceptueel verkeerd en nutteloos.

Status: GEREPAREERD (verwijderd naar alleen <|endoftext|>)

Fout 5: [UNK] in HTML-Rapport Niet Consequent (DOCUMENTATIE)

Locatie: Stap2_BPE_Tokenizer_Rapport.html, "Geconfigureerde Speciale Tokens"

Het probleem:

Het HTML-rapport noemde [UNK] in de uitleg ("hoewel Byte-Level BPE dit zelden nodig heeft") terwijl de Python-code het eigenlijk gebruikt. Dit is inconsistent en verwarrend.

Status: GEREPAREERD (verwijderd uit beide plaatsen)

✅ Reparaties Ingepast

Reparatie 1: Merge-Extractie Logica

Aanpassing:

if hasattr(tokenizer.model, 'merges') and tokenizer.model.merges:
    with open(merges_path, 'w') as f:
        for merge_pair in tokenizer.model.merges:
            f.write(f"{merge_pair}\n")
else:
    logger.warning("No merges found...")

• Controleert eerst of merges bestaan
• Behandelt elk merge_pair als string (niet als tuple)
• Geeft graceful warning als extraction niet mogelijk is

Reparatie 2: Nederlands Taalgebruik

Aanpassingen in code:

  • "Vocab size" → "Vocabulaire grootte"
  • "Gemiddeld tokens" → "Gemiddeld aantal tokens"
  • "Vocab size: 16,000" → "Vocabulaire grootte: 16.000" (ook juiste scheidingsteken)

📚 Lessen uit deze Verificatie

  • Verificatie is makkelijk om oppervlakkig te doen: Een checklist met "✓" zeggen is makkelijk. Echt testen is moeilijker.

  • Self-review is blind: Ik miste deze fouten omdat ik dezelfde aannames maakte. Externe feedback was nodig.
  • Eerlijkheid over volume: "18/18" klinkt goed, maar "14/18 + 4 gefixeerd" is veel waarheidsgetrouwer.
